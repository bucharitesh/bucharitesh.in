## Props

| Prop           | Type                    | Description                                          | Default     |
|----------------|-------------------------|------------------------------------------------------|-------------|
| icon           | number[][]              | 2D array representing the icon's pixel layout        | (Required)  |
| baseColor      | string                  | Base color of the icon                               | (Required)  |
| flickerColor   | string                  | Color used for the flicker effect                    | (Required)  |
| secondaryColor | string                  | Secondary color for additional details               | "gray"      |
| size           | number                  | Size of the icon in pixels                           | 80          |
| flickerChance  | number                  | Probability of pixel color change on hover (0 to 1)  | 0.3         |
| pixelShape     | "circle" \| "square"    | Shape of individual pixels                           | "circle"    |
| className      | string                  | Additional CSS classes for the icon container        |             |

## Usage

```jsx
import PixelIcon from './PixelIcon'

const MyComponent = () => {
  const iconData = [
    [1, 1, 1],
    [1, 0, 1],
    [1, 1, 1],
  ]

  return (
    <PixelIcon
      icon={iconData}
      baseColor="#ff0000"
      flickerColor="#ffff00"
      secondaryColor="#0000ff"
      size={100}
      flickerChance={0.5}
      pixelShape="square"
    />
  )
}
```

## Features

- **Interactive Hover Effect**: The icon's pixels flicker and change colors when hovered over.
- **Customizable Colors**: Set base, flicker, and secondary colors to match your design.
- **Adjustable Size**: Easily change the icon's size to fit your layout.
- **Pixel Shape Options**: Choose between circular or square pixels.
- **Performance Optimized**: Uses `requestAnimationFrame` for smooth animations.
- **Responsive**: Adapts to different screen resolutions using `devicePixelRatio`.

## Accessibility

The component includes appropriate ARIA attributes for better accessibility:

- `role="img"` on the container div
- `aria-label="Pixel icon"` for screen readers
- `aria-hidden="true"` on the canvas element
